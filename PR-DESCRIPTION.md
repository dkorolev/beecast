## Summary
- Extract the HTML-page pipeline into `beecast-page`, a new **zero-dependency** library crate (`page/`), so other Rust tools — `scsh`, which deliberately keeps a tiny dependency set and hand-rolls its JSON, is the motivating consumer — can render an asciinema `.cast` into a self-contained offline `.html` page without shelling out to the `beecast` CLI or pulling serde/schemars/anyhow transitively.
- The `beecast` CLI keeps its exact external behavior and output: for the same inputs, the generated HTML is **byte-identical** to what the serde-era renderer produced, and the test suite now pins that.
- `beecast-dto` stays untouched; the workspace grows to three published crates (`beecast-dto`, `beecast-page`, `beecast`), all inheriting the one workspace version.

## What This Changes
The page pipeline — `page.html`, the vendored asciinema-player, the token-substitution renderer with its escaping helpers, and the cast-inspection logic (`CastInfo`, `CastError`, `inspect`) — moves from `cli/src/` into the new `page/` crate. Its public API takes plain strings and floats, never serde types: `PageMeta { title: Option<&str>, summary: Option<&str>, chapters: &[(f64, &str)] }`, `build_page(cast_ndjson, &meta, fallback_title) -> String`, and `inspect(content) -> Result<CastInfo, CastError>`.

To keep the crate at zero dependencies, the serde_json and thiserror usages were replaced within the crate by std-only equivalents that preserve the serde semantics exactly: a strict JSON reader for cast inspection (full grammar, surrogate-pair escapes, duplicate keys last-wins, serde's `as_u64` contract and 128-level nesting cap), a writer whose string escaping and float notation reproduce serde_json's output byte-for-byte (ryu's decimal/scientific window, `.0` tails, `\u00xx` control escapes), and hand-written `Display`/`Error` impls for `CastError`. The embedded metadata document is serialized by hand from the plain-strings input with the CLI's exact escaping semantics, including the `\u003c` neutralization of `<` inside `<script>`.

The CLI converts its validated serde `CastMeta` into the plain-strings shape at the boundary and drops its direct `thiserror` dependency. The vendored player (Apache-2.0) now ships in `beecast-page`, whose SPDX license is therefore `MIT AND Apache-2.0`, same as the CLI's, which keeps that expression since every binary and generated page still embeds the player.

## How to Verify
- **Deterministic gate** (same as CI and pre-push): `cargo fmt --check && cargo clippy --workspace --all-targets -- -D warnings && cargo test --workspace && cargo test --workspace --release && python3 -m unittest discover -s seecast/tests`.
- **Byte-identity, pinned in the suite**: `generated_page_is_byte_identical_to_the_serde_era_renderer` in `cli/tests/cli.rs` renders the fixture recording (with and without its sidecar) through the real binary and asserts FNV-1a fingerprints captured from the CLI at the commit before the extraction. During the refactor the outputs for three inputs — the fixture pair, a bare recording, and a hostile cast/sidecar with `</script>` payloads, control characters, and boundary floats (`1e-5`, `1e15`, `1e16`, `1.23e17`) — were also compared with `cmp` against pre-refactor captures: identical.
- **Differential against serde itself**: `cli/tests/parity.rs` compares the metadata document embedded by `beecast_page::build_page` with `serde_json::to_string` of the same `CastMeta`, over hostile strings and ~5000 pseudo-random finite `f64` bit patterns. It lives in the CLI's suite so `beecast-page` keeps zero dependencies, dev-dependencies included.
- **Zero dependencies**: `cargo tree -p beecast-page` prints the one line.
- **Publish readiness**: `cargo publish -p beecast-page --dry-run` verifies offline; `cargo package -p beecast-page --list` shows the crate ships its sources, the template, the vendored player with its Apache-2.0 text, the README, and the MIT license.

## Implementation Details
- **`page/src/json.rs`**: the std-only JSON reader and writer. The reader mirrors `serde_json::from_str` strictness (escape validation incl. UTF-16 surrogate pairs, `1e999` rejected, depth cap 128); parsed string *values* and booleans carry no payload since `inspect` only reads numbers, arrays, and object keys. The writer's `fmt_f64` takes the shortest round-trip digits from `{:e}` and reassembles ryu's notation (decimal while the point lands in `(-5, 16]`, integral `.0` tails, `1e+16`-style scientific, `null` for non-finite — serde's own renderings, pinned value-by-value in unit tests).
- **`page/src/cast.rs`**: moved verbatim apart from the JSON swap and the hand-written `CastError` impls; a new test pins the serde-era reading that a fractional `"version": 2.0` is not an asciicast.
- **`page/src/page.rs`**: `build_page` unchanged in structure; `meta_json` hand-serializes `PageMeta` exactly as serde serialized `CastMeta` (declaration order, absent/empty fields omitted, `{}` when nothing is set). The footer version is inherited from the shared `[workspace.package]`, so it is still the CLI's version.
- **`cli/`**: `run_build` converts `CastMeta` → `PageMeta` (borrowed strings) after validation; `cli/src/{cast.rs,page.rs,page.html,vendor/}` are gone; machine-mode JSON documents still use serde_json.
- **Docs**: workspace `README.md` table and license note, `CONTRIBUTING.md` (four parts, re-vendoring paths, re-pinning note), `PUBLISHING.md` (publish `beecast-dto` and `beecast-page` in either order, then `beecast`; the version now lives in three places), and the new `page/README.md`.
